# Sample Data

This folder contains sample datasets used throughout the workshop.

---

## Available Files

| File | Description | Rows |
|------|-------------|------|
| `Products.csv` | Product catalog (outdoor/sporting goods) | 65 |
| `Stores.csv` | Store locations (Pacific Northwest) | 12 |
| `Calendar.csv` | Date dimension (2023-2025) | 1,096 |
| `Targets.csv` | Monthly revenue targets by store | ~432 |
| `SupplierCosts.csv` | Actual supplier invoice costs | 65 |

> **Note:** Sales transactions are generated dynamically by the notebook `notebooks/01-Generate-Sales-Data.ipynb` to allow configurable data volumes.

---

## Data Generation

The `generate_sales.py` script (and the notebook version) creates realistic sales data with:
- Seasonal patterns (holiday spikes, summer outdoor gear)
- Regional variations by store
- Product category mix variations
- Configurable transaction volume (default: 100,000)

---

## Sample Data Schema

### Products
| Column | Type | Description |
|--------|------|-------------|
| ProductID | STRING | Primary key (e.g., "1001") |
| ProductName | STRING | Product name |
| Category | STRING | Product category |
| SubCategory | STRING | Product subcategory |
| UnitPrice | DECIMAL | Retail price |
| UnitCost | DECIMAL | Budgeted cost |
| IsActive | BOOLEAN | Active product flag |

### Stores
| Column | Type | Description |
|--------|------|-------------|
| StoreID | STRING | Primary key (e.g., "S01") |
| StoreName | STRING | Store name |
| City | STRING | City |
| State | STRING | State (WA, OR, ID, MT) |
| Region | STRING | Region name |
| OpenDate | DATE | Store opening date |
| SquareFeet | INT | Store size |
| ManagerName | STRING | Store manager |

### Calendar
| Column | Type | Description |
|--------|------|-------------|
| Date | DATE | Primary key |
| Year | INT | Calendar year |
| Quarter | INT | Quarter (1-4) |
| Month | INT | Month (1-12) |
| MonthName | STRING | Month name |
| Day | INT | Day of month |
| DayOfWeek | INT | Day of week (1-7) |
| DayName | STRING | Day name |
| WeekOfYear | INT | Week number |
| IsWeekend | BOOLEAN | Weekend flag |
| FiscalYear | INT | Fiscal year |
| FiscalQuarter | INT | Fiscal quarter |

### Targets
| Column | Type | Description |
|--------|------|-------------|
| StoreID | STRING | Foreign key to Stores |
| Year | INT | Target year |
| Month | INT | Target month |
| RevenueTarget | DECIMAL | Monthly revenue target |
| TransactionTarget | INT | Monthly transaction target |

### SupplierCosts
| Column | Type | Description |
|--------|------|-------------|
| ProductID | STRING | Foreign key to Products |
| SupplierID | STRING | Supplier identifier |
| SupplierName | STRING | Supplier company name |
| SupplierCost | DECIMAL | Actual invoiced cost |
| LeadTimeDays | INT | Delivery lead time |
| MinOrderQty | INT | Minimum order quantity |
| LastUpdated | DATE | Last price update date |

### Sales (Generated by Notebook)
| Column | Type | Description |
|--------|------|-------------|
| SalesID | STRING | Primary key |
| TransactionDate | DATE | Transaction date |
| ProductID | STRING | Foreign key to Products |
| StoreID | STRING | Foreign key to Stores |
| Quantity | INT | Units sold |
| UnitPrice | DECIMAL | Sale price |
| Discount | DECIMAL | Discount percentage |
| TotalAmount | DECIMAL | Net sale amount |
| TotalCost | DECIMAL | Cost of goods sold |

## Data Volume Recommendations

| Scenario | Sales Rows | Notes |
|----------|-----------|-------|
| Quick demo | 10,000 | Fast processing, minimal wait |
| Workshop | 100,000 | Good balance of realism and speed |
| Scale demo | 500,000+ | Shows Direct Lake benefits |

Adjust `NUM_TRANSACTIONS` in the notebook configuration cell.
